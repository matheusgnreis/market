Informações do aplicativo Essas informações serão usadas para identificar seu aplicativo.
Urls Abaixo você encontra informação sobre as urls. Url de redirecionamento, deve ser informada caso seu aplicativo necessite de autenticação de algum serviço externo, ou até mesmo um oauth próprio. Autenticação deve ser informada caso seu aplicativo consuma recursos da API E-com.plus. Caso seu aplicativo utilize autenticação, será necessário informar a url onde receberá callbacks de autenticação enviados pela API E-com.plus. Se o aplicativo utiliza autenticação, você pode selecionar quais recursos seu aplicativo irá consumir na API para solicitar permissão do usuário na instalação do aplicativo no market. Você pode informar um site oficial da sua aplicação. Também pode informar o repositório do aplicativo no Github.

Icone do Aplicativo Os ícones ajudam os lojistas a reconhecer e identificar seu aplicativo.
Screenshot Screenshot oferece um preview do comportamento do seu aplicativo instalado na loja ou pode oferecer um detalhe visual das opções e funcionalidades do aplicativo aplicadas na store.
Descrições A descrição do seu aplicativo pode ser decisiva para levar um lojista a usá-lo ou não. A descrição curta é extremamente importante e obrigatório, é a descrição que será exibida na página inicial do market e nos resultados de busca. A descrição longa é tão importante quando a curta, é onde será exibido todas as opções e funcionalidades do seu aplicativo. Não poupe palavras, nesse campo é possível usar sintaxe Markdown.
Sobre o aplicativo Essa informações são usadas pelo nosso saas para entender o funcionamento do seu aplicativo em nossa plataforma.
Módulos É possível que seu aplicativo faça uso de algum módulo de nossa API para funcionar, temos uma infinidade de módulos disponíveis para facilitar o desenvolvimento de novas funcionalidades ou incrementar funcionalidades ja existentes. Seja para realizar pagamento, cálcular frete, aplicar desconto, criar transações e muito mais. A definição de cada módulo pode ser encontrada aqui e seu aplicativo deve respeitar os schemas de requisição e resposta de cada módulo que ele irá se comunicar para funcionar corretamente e ser aprovado em nossa homologação.
Configurações do Lojista Aqui você pode configurar opções que serão preenchidas pelo lojista. É aqui que você define as propriedades que estaram presentes no DATA e no HIDDEN_DATA do seu aplicativo.
